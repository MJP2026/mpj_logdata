Goal:
KI should be used for analysing a huge amount of log data.

With help from KI the kibana log data should,
be analysed to get a quick overview about the system status.

The successful and the failed requests should be shown quantified.

The error message should be classified and quantified.

The most frequent error messages should be listed.
The details should be shown in a separate view/file/link.

The data should be processed only on the local machine.
No upload from log data to any KI server outside the local machine.

The log data should be treated confidentially.

The passwords and token (sensitive data) should be masked.
